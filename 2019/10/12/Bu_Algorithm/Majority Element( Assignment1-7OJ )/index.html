Majority Element( Assignment1-7OJ )

Problem description

Given an array of size n, find the majority element. The majority element is the element that appears more than $\lfloor n/2 \rfloor$ times.

You may assume that the array is non-empty and the majority element always exist in the array.

The range of length of array is n(1≤n≤5,000,000) and the element (integer) in array is no bigger than 10,000,00000.

Input

Line 1: the length of array.

Line 2: the all elements in array and split by spaces

Output

Line 1: A single integer that is the majority element.

找出数组中最多的那个值，而且该值的数量大于数组实际数据长度的一半。
有三种方法去想这个问题。

一、最直接想法

如果数组是排好序列的，直接输出中间值。

该方法首先对数组进行快排，然后再找到数组的中间值，虽然比较好想，但是时间复杂度较高 $O(nlogn)$，因为快排 $T(n) = 2T(n/2) + O(n) = O(nlogn)$ ，挖个坑，详细步骤见后面谋篇文章。

二、改进

由于在 topk 问题中，有 O(n) 算法可以得到数组中任意第 k 大的数字。所以可以在数组中任意选取一个数字，为了省事，我就直接去了最后一个数，其实这种方法并不好，容易造成较差的复杂度，建议随机取。这里不建议采用卜老师上课讲的 $[\frac{n}{4},\frac{3n}{4}]$ 区间查找，因为数组中的数不是完全独立的，比如 $\{2, 2, 1, 1, 2, 1, 1\}$。关于卜老师的这个选取 pivot 的方法，后面找机会补上。由于我们不需要将数组进行全部按序排列，因为只是要得到数组的中位数就行。

选好随机数之后调整顺序，比它小的在左边，大的在右边，如果排序后下标刚好n/2，那么该数就是中位数。若大于n/2，那么中位数在它左边，反之在右边。用这种方法递归时只递归一半，所以总的时间复杂度是 $T(n) = T(n/2) + O(n) = O(n)$。

三、根据数组特点

由于 Majority Element 是数组中数目最多的数，那么其他的数出现的总和还不如一半。在遍历数组时，可以保存两个值，一个是数组的一个元素的值，一个是该值出现的次数。

如果连续出现一个数，那么该数出现的次数一直加，直到出现其他数时，该计数值开始减，直到减到 0 为止。然后下一个数开始加一。由于 Majority Element 最多，所以遍历结束后，如果计数值不为0时的元素，一定是 Majority Element。

因为只执行一次循环遍历，所以时间复杂度是 $T(n) = O(n)$。
